# Projection Is Not Authority

## Core Rule

```
Projection is not authority.
Notation is not execution.
Interpretation is not acceptance.
```

## What This Means

A projection presents a view. It does not decide truth. A notation
declares a reading. It does not execute a relation.

## Prohibited Equivalences

```
notation        != execution
projection      != authority
interpretation  != acceptance
declaration     != proof
canonical doc   != runtime behavior
```

No layer may impersonate another.

## The Boundary

```
OMI defines the object model.
SETCO classifies how other things relate to it.
No layer may impersonate another.
```

## One Sentence

Projection presents what has been computed. It does not compute.
Validation accepts. Projection is not authority.
