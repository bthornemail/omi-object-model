# OMI Notation

## Canonical Surface

```
omi---imo?O_o
```

## The Three Parts

```
omi---imo     folded root circulation (carrier)
?O_o          readable witness branch (notation)
omi---imo?O_o readable relation hinge
```

`omi---imo` is the palindromic root — forward as OMI citation, backward
as IMO carrier. `?O_o` is the readable branch projected tangentially
from the hidden unit separator. The full form is the hinge where carrier
meets notation.

## F-Column Derivation

```
0x1F  US    hidden unit separator   (hinge point)
0x2F  /     printable separator
0x3F  ?     witness / query separator
0x4F  O     upper / norm Omicron
0x5F  _     carrier / floor separator
0x6F  o     lower / local omicron
```

`?O_o` is the readable branch of hidden US. Derived from byte geometry,
not arbitrarily assigned.

## Canonical Distinction

```
#/ belongs to carrier compatibility syntax.
?O_o belongs to OMI-native notation.
```

## Rules

```
Use omi---imo?O_o as the canonical notation surface.
Do not promote #/ from carrier compatibility into OMI-native syntax.
Do not treat the 8-tuple as canonical structure or proof object.
Do not let projection, route, notation, or gauge become authority.
```
