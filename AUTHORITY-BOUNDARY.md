# Authority Boundary

## Core rule

```
OMI defines the object model.
SETCO classifies how other things relate to it.
No layer may impersonate another.
```

## What OMI may do

```
define notation
declare objects
bind addresses
compute relation witnesses
normalize and close
```

## What OMI may not do

```
classify external environments
stage proposals for external tools
adjudicate promotion status
teach external relationships
```

## Role boundaries

```
Omicron           = deterministic normalization and closure surface
Omnicron          = resolver and projection surface
Metatron          = balanced alternating witness / scribe
Tetragrammatron   = lawful fold governor / validator
```

## Prohibited equivalences

```
notation          != execution
projection        != authority
interpretation    != acceptance
declaration       != proof
canonical doc     != runtime behavior
```
